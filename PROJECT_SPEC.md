# KreativIT Website Specification (Version 1)

## Project Overview

**Project Name:** KreativIT Website v1 (Project November)

### Primary Goal

Build a professional website that helps attract freelance clients before the November visa renewal.

Every design and development decision should support this goal.

**Project Rule:**

> Progress over perfection.

If a feature does not meaningfully increase the chances of getting a client before November, it belongs in the Version 2 backlog.

# Design Direction

The website should feel:

* Bright
* Warm
* Modern
* Minimal
* Professional
* Calm

Avoid:

* Stock photography
* Generic tech illustrations
* Overly flashy animations
* Unnecessary visual clutter

The design philosophy should prioritize:

* Excellent typography
* Consistent spacing
* Strong visual hierarchy
* Accessibility
* Simplicity over decoration

Design first, then build. 

Establish the design system before implementing additional sections whenever practical.

A simple, cohesive design system should be established before building additional sections.

---

# Writing Style

All copy should be:

* Concise
* Easy to scan
* Client-focused
* Free of unnecessary jargon

The website should answer three questions within 30 seconds:

1. What do I do?
2. Can I solve the visitor's problem?
3. How can they contact me?

---

# Technical Direction

Preferred stack:

* React
* TypeScript
* Vite
* Tailwind CSS
* Responsive design
* Mobile-first
* Fast loading
* Accessible
* SEO-friendly

Architecture principles:

* Components should remain presentational.
* Site content should live outside components.
* Configuration belongs in `config/`.
* Reusable styling belongs in `styles/`.
* Data should live in `data/`.

---

# Development Workflow

The project follows this workflow:

> Build → Verify → Commit → Push

Commits should be:

* Small
* Focused
* Easy to understand

Use Conventional Commits:

* feat:
* fix:
* refactor:
* style:
* docs:
* chore:

Avoid large commits containing unrelated work.

Always verify changes in the browser before committing.

---

# AI Collaboration Guidelines

When working on this project:

* Work one step at a time.
* Keep answers short and concise.
* Avoid unnecessary technical jargon.
* Explain concepts in plain language.
* Prefer many small changes over large rewrites.
* Always ask for the latest version of a file before editing it if the user may have made independent changes.
* Never assume previously shared files are still current.
* Keep track of architecture decisions, design decisions, backlog items, and commit boundaries.
* Recommend starting a new Project November chat before the conversation becomes too large.
* Before switching chats, provide a short project handoff including:

  * Completed work
  * Current architecture
  * Latest commit
  * Important decisions
  * Backlog updates
  * Next recommended task
